{{/*
oms web-service:
  Template to run modelName_mpi executable on Linux using OpenMPI 4.x

Use this template if you experiencing CPU binding problems, which may be common on CentOS 8 clusters

Rename this file into:
  mpi.ModelRun.template.txt
to use it by default to run models on MPI cluster.

Oms web-service process this template as:
  - skip empty lines
  - use first non-empty line as executable to run
  - substitute template arguments
  - combine result into single line
after template processing oms run it as: exec.Cmd.Start()

Arguments of template:
struct {
  ModelName string            // model name
  ExeStem   string            // base part of model exe name, usually modelName
  Dir       string            // work directory to run the model
  BinDir    string            // bin directory where model exe is located
  MpiNp     int               // number of MPI processes
  Args      []string          // model command line arguments
  Env       map[string]string // environment variables to run the model
}

Example of result:
  mpirun -n 16 --bind-to none -wdir models/bin -x key=value ./modelName_mpi -OpenM.LogToFile true

*/}}

mpirun
-n
{{ .MpiNp}}
--bind-to none
{{with .Dir}}
-wdir
{{.}}
{{end}}
{{range $key, $val := .Env}}
-x
{{$key}}={{$val}}
{{end}}
{{.BinDir}}/{{.ExeStem}}_mpi
{{range .Args}}
{{.}}
{{end}}
